## 임베딩 시각화

임베딩 모델은 각 이미지에 대해 고차원 벡터(예: 512 또는 768차원)를 생성합니다. 이미지들이 어떻게 군집을 이루는지 시각화하기 위해, ssook은 세 가지 알고리즘 중 하나를 사용하여 이 벡터를 2D로 축소합니다.

### t-SNE

**t-Distributed Stochastic Neighbor Embedding (t-분포 확률적 이웃 임베딩)**

**동작 원리:**
1. 원래의 고차원 공간에서 가우시안 분포를 사용하여 쌍별 유사도를 계산합니다. 가까운 점은 높은 유사도를, 먼 점은 낮은 유사도를 갖습니다. **Perplexity** 파라미터(일반적으로 5~50)는 각 점이 고려하는 이웃의 수를 제어합니다.
2. 2D 대상 공간에서 Student's t-분포(가우시안보다 꼬리가 두꺼움)를 사용하여 쌍별 유사도를 계산합니다.
3. 두 유사도 분포 간의 **KL Divergence** (차이 측정 지표)를 경사 하강법으로 최소화합니다.
4. t-분포의 두꺼운 꼬리가 "밀집 문제(crowding problem)"를 방지합니다 — 2D에서 적당히 먼 점들이 함께 뭉치는 것을 막아줍니다.

**특성:**
- ✅ 지역적 군집 구조를 드러내는 데 탁월
- ✅ 클래스 간 시각적으로 명확한 분리
- ⚠️ 전역 거리는 의미가 없음 — 군집 간 거리가 실제 유사도를 반영하지 않음
- ⚠️ 비결정적 — 실행할 때마다 다른 레이아웃이 생성될 수 있음
- ⚠️ 대규모 데이터셋에서 느림 (O(n²) 복잡도)

### UMAP

**Uniform Manifold Approximation and Projection (균일 다양체 근사 및 투영)**

**동작 원리:**
1. 고차원 공간에서 가중 k-최근접 이웃 그래프를 구축합니다. 각 점은 가장 가까운 이웃들과 연결되며, 엣지 가중치는 거리에 기반합니다.
2. 2D 대상 공간에서 유사한 그래프를 구성합니다.
3. 교차 엔트로피 손실을 사용하여 두 그래프가 최대한 유사하도록 2D 레이아웃을 최적화합니다.
4. 수학적 기반은 위상학적입니다 — UMAP은 데이터의 "형태" (다양체 구조)를 보존합니다.

**특성:**
- ✅ t-SNE보다 지역적, 전역적 구조를 모두 더 잘 보존
- ✅ t-SNE보다 빠르며, 더 큰 데이터셋으로 확장 가능
- ✅ 실행 간 더 결정적
- ⚠️ `umap-learn` 패키지가 필요합니다

### PCA

**Principal Component Analysis (주성분 분석)**

**동작 원리:**
1. 각 차원의 평균을 빼서 데이터를 중심화합니다.
2. 중심화된 데이터의 공분산 행렬을 계산합니다.
3. 공분산 행렬의 고유벡터(주성분)를 찾습니다. 첫 번째 고유벡터는 최대 분산 방향을, 두 번째는 (첫 번째에 직교하는) 나머지 최대 분산 방향을 가리킵니다.
4. 상위 2개 고유벡터에 데이터를 투영하여 2D 좌표를 얻습니다.

**특성:**
- ✅ 결정적 — 동일한 입력은 항상 동일한 출력을 생성
- ✅ 매우 빠름 (선형대수, 반복 없음)
- ✅ 전역 구조가 보존됨 — 거리가 의미 있음
- ⚠️ 선형 관계만 포착 — 복잡한 군집 구조를 놓칠 수 있음
- ⚠️ 분산이 여러 차원에 분산되어 있으면 2D 투영이 뭉개져 보일 수 있음

**ssook에서 임베딩 시각화 사용하는 방법:**
1. **Analysis** 탭 → **Embedding Visualization**으로 이동
2. ONNX 특징 추출 모델 로드
3. 이미지 디렉토리 설정 (폴더 구조: `class_name/image.jpg`)
4. 알고리즘 선택 (t-SNE / UMAP / PCA)
5. **Run** 클릭 — 클래스별로 색상이 지정된 2D 산점도가 생성됩니다

## CLIP Zero-Shot 분류

CLIP (Contrastive Language-Image Pre-training)은 태스크별 학습 데이터 없이 텍스트 설명을 사용하여 이미지를 분류할 수 있게 합니다.

**동작 원리:**

1. **이중 인코더 아키텍처**: CLIP은 두 개의 별도 신경망으로 구성됩니다:
   - **이미지 인코더**: 이미지를 고정 크기 임베딩 벡터로 변환
   - **텍스트 인코더**: 텍스트 설명을 동일한 크기의 임베딩 벡터로 변환

2. **이미지 전처리**: 입력 이미지를 인코더의 예상 해상도(일반적으로 224×224)로 리사이즈하고, RGB로 변환하며, ImageNet 통계(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])로 정규화한 후 NCHW 텐서로 배열합니다.

3. **Zero-Shot 분류 흐름**:
   - 입력 이미지 인코딩 → 이미지 임베딩
   - 각 후보 텍스트 레이블 인코딩 (예: "a photo of a cat", "a photo of a dog") → 텍스트 임베딩
   - 이미지 임베딩과 각 텍스트 임베딩 간의 **코사인 유사도** 계산:
     ```
     similarity = (image_emb · text_emb) / (‖image_emb‖ × ‖text_emb‖)
     ```
   - 유사도가 가장 높은 텍스트 레이블이 예측 클래스가 됩니다

4. **작동하는 이유**: 사전 학습 과정에서 CLIP은 이미지와 텍스트 설명을 공유 임베딩 공간에서 정렬하는 것을 학습했습니다. 고양이 이미지는 이 공간에서 "a photo of a cat" 텍스트와 가깝게 위치합니다.

**ssook에서 사용하는 방법:**
1. **CLIP** 탭으로 이동
2. 이미지 인코더 ONNX 모델과 텍스트 인코더 ONNX 모델 로드
3. 후보 텍스트 레이블 입력 (한 줄에 하나씩)
4. 이미지 또는 이미지 세트 로드
5. **Run** 클릭 — 각 레이블에 대한 유사도 점수가 표시됩니다

## Embedder 평가

Embedder 평가는 특징 추출 모델이 임베딩 공간에서 서로 다른 클래스를 얼마나 잘 분리하는지 측정합니다.

**동작 원리:**

1. **데이터셋 구조**: `class_name/image.jpg` 형태로 이미지를 구성합니다. 각 폴더 이름이 클래스 레이블입니다.
2. **임베딩 추출**: 각 이미지를 ONNX 모델에 통과시켜 임베딩 벡터를 생성합니다.
3. **L2 정규화**: 모든 임베딩을 단위 길이로 정규화하여, 코사인 유사도가 내적과 같아지도록 합니다.
4. **유사도 행렬**: 쿼리와 갤러리 임베딩 간의 전체 Q×G 코사인 유사도 행렬을 계산합니다.
5. **지표**:
   - **Retrieval@1**: 각 쿼리에 대해 가장 유사한 상위 1개 갤러리 이미지가 같은 클래스인가?
   - **Retrieval@K**: 각 쿼리에 대해 상위 K개 결과에 올바른 클래스가 포함되어 있는가?
   - **평균 코사인 유사도**: 각 쿼리의 최대 유사도 평균 (매칭의 확신도)

**다중 이미지 비교**: 여러 이미지를 선택하여 쌍별 코사인 유사도 행렬을 볼 수도 있습니다. 특정 이미지들이 임베딩 공간에서 유사한지 다른지 확인하는 데 유용합니다.

**ssook에서 사용하는 방법:**
1. **Evaluation** 탭 → **Embedder**로 이동
2. 특징 추출 ONNX 모델 로드
3. 데이터셋 디렉토리 설정
4. Retrieval@K의 K 값 설정 (기본값: 5)
5. **Run** 클릭
