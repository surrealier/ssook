## Object Tracking (객체 추적)

객체 추적은 비디오 프레임에 걸쳐 탐지된 객체에 일관된 ID를 부여합니다. ssook은 모든 탐지 모델 위에서 작동하는 두 가지 추적 알고리즘을 제공합니다.

### SORT

**Simple Online and Realtime Tracking**

SORT는 IoU 기반의 간단한 트래커입니다:

1. **탐지**: 탐지 모델이 현재 프레임의 바운딩 박스를 생성합니다.
2. **IoU 매칭**: 기존 트랙의 마지막 위치와 모든 새 탐지 간의 IoU(겹침)를 계산합니다. M×N 비용 행렬이 생성됩니다.
3. **탐욕적 할당**: IoU가 가장 높은 쌍부터 트랙과 탐지를 매칭합니다. 매칭에는 IoU ≥ `iou_threshold` (기본값: 0.3)가 필요합니다.
4. **매칭된 트랙 업데이트**: 트랙의 위치를 업데이트하고, 히트 카운트를 증가시키며, 미업데이트 시간을 초기화합니다.
5. **새 트랙 생성**: 매칭되지 않은 탐지는 새 ID를 가진 새 트랙이 됩니다.
6. **미매칭 트랙 에이징**: 이번 프레임에서 매칭되지 않은 트랙은 미업데이트 시간이 증가합니다. `max_age` 프레임 동안 매칭 없이 지속된 트랙은 제거됩니다.
7. **출력**: 총 매칭 횟수가 `min_hits` 이상인 트랙만 보고됩니다 (단일 프레임 탐지로 인한 깜빡임 방지).

각 트랙은 시각화를 위해 최근 60개 중심점으로 구성된 **궤적(trajectory)**을 유지합니다.

### ByteTrack

ByteTrack은 **2단계 연관(two-stage association)**을 사용하여 저신뢰도 탐지를 복구함으로써 SORT를 개선합니다:

**1단계 — 고신뢰도 매칭:**
1. 탐지를 **고신뢰도** (점수 ≥ `high_thresh`, 기본값: 0.5)와 **저신뢰도** (점수 ≥ `low_thresh`, 기본값: 0.1)로 분리합니다.
2. 모든 기존 트랙을 고신뢰도 탐지와 IoU로 매칭합니다 (SORT와 동일).
3. 매칭된 트랙을 업데이트하고, 미매칭 트랙을 수집합니다.

**2단계 — 저신뢰도 복구:**
4. **남은 미매칭 트랙**을 저신뢰도 탐지와 IoU로 매칭합니다.
5. 이를 통해 일시적으로 가려지거나 탐지 신뢰도가 낮았던 트랙을 복구합니다.
6. 두 단계 모두에서 매칭되지 않은 트랙은 미업데이트 시간이 증가합니다.

**새 트랙 생성:**
7. 매칭되지 않은 **고신뢰도** 탐지만 새 트랙을 생성합니다. 저신뢰도 탐지만으로는 새 트랙을 시작하지 않습니다 (노이즈 방지).

**ByteTrack이 더 나은 이유:**
- 일반 트래커는 저신뢰도 탐지를 완전히 버려서, 객체가 부분적으로 가려질 때 트랙이 끊어집니다.
- ByteTrack은 저신뢰도 탐지를 기존 트랙 유지에 활용하되, 새 트랙 생성에는 사용하지 않습니다 (오탐 방지).

### 트래킹 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `max_age` | 30 | 매칭 없이 트랙을 유지하는 프레임 수 |
| `min_hits` | 3 | 트랙이 보고되기 위한 최소 매칭 횟수 |
| `iou_threshold` | 0.3 | 유효한 매칭을 위한 최소 IoU |
| `high_thresh` | 0.5 | ByteTrack: 1단계 매칭의 최소 신뢰도 |
| `low_thresh` | 0.1 | ByteTrack: 2단계 매칭의 최소 신뢰도 |

**ssook에서 사용하는 방법:**
1. **Viewer** 탭으로 이동
2. 탐지 모델을 로드하고 비디오 열기
3. 트래킹을 활성화하고 알고리즘 선택 (ByteTrack 또는 SORT)
4. 객체에 지속적인 ID와 궤적 트레일이 부여됩니다

## Smart Sampler (스마트 샘플러)

대규모 데이터셋을 다룰 때 대표적인 부분집합이 필요한 경우가 많습니다. 스마트 샘플러는 세 가지 전략을 제공합니다.

### Balanced Sampling (균형 샘플링)

**목표**: 모든 클래스의 균등한 대표성과 각 클래스 내 공간적 다양성 확보.

**동작 원리:**
1. 모든 레이블 파일을 파싱하여 클래스 → 이미지 맵을 구축합니다.
2. 목표 수를 클래스 수로 균등 분배합니다: `per_class = target / num_classes`.
3. 각 클래스에 대해 풀이 `per_class`보다 크면 **최원점 샘플링(farthest-point sampling)**을 사용합니다:
   - 각 이미지의 특징을 계산합니다: 평균 바운딩 박스 중심 `[cx, cy]` (0~1 정규화).
   - 랜덤 시드 이미지로 시작합니다.
   - 이미 선택된 모든 이미지로부터 **가장 먼** 이미지를 반복적으로 선택합니다 (유클리드 거리 제곱 사용).
   - 이를 통해 공간적 다양성을 보장합니다 — 선택된 이미지가 한 영역에 몰리지 않고 이미지 공간의 다양한 영역을 커버합니다.
4. 풀이 `per_class`보다 작으면 해당 클래스의 모든 이미지를 가져옵니다.

**사용 시기**: 클래스 불균형이 있고 균등한 대표성이 필요하거나, 다양한 공간적 커버리지가 필요할 때.

### Stratified Sampling (층화 샘플링)

**목표**: 원래 클래스 분포를 더 작은 규모로 유지.

**동작 원리:**
1. 각 클래스의 비율을 계산합니다: `class_count / total_associations`.
2. 각 클래스에서 `proportion × target`개의 이미지를 랜덤으로 샘플링합니다.
3. 총 수가 목표보다 적으면, 미선택 풀에서 랜덤 이미지로 나머지를 채웁니다.

**사용 시기**: 원래 분포를 통계적으로 대표하는 더 작은 데이터셋이 필요할 때.

### Random Sampling (랜덤 샘플링)

**목표**: 단순한 비편향 랜덤 부분집합.

**동작 원리**: 재현성을 위해 고정 시드로 `random.sample(all_images, target)`을 수행합니다.

**사용 시기**: 클래스 균형이 중요하지 않거나, 베이스라인으로 사용할 때.

**ssook에서 사용하는 방법:**
1. **Data** 탭 → **Smart Sampler**로 이동
2. 이미지 디렉토리, 레이블 디렉토리, 출력 디렉토리 설정
3. 전략 선택 (Balanced / Stratified / Random)
4. 목표 수와 랜덤 시드 설정
5. **Run** 클릭 — 샘플링 전후 클래스 분포 테이블이 표시됩니다

## Near-Duplicate Detection (유사 중복 탐지)

유사 중복 탐지는 **dHash** (difference hash)를 사용하여 지각적으로 유사한 이미지(바이트 단위로 동일하지 않더라도)를 찾습니다.

**dHash 동작 원리:**

1. 이미지를 9×8 픽셀(그레이스케일)로 **리사이즈**합니다. 고주파 디테일을 제거하고 크기를 정규화합니다.
2. **수평 그래디언트 계산**: 각 행에서 인접 픽셀을 비교합니다. 왼쪽 픽셀이 오른쪽보다 밝으면 1, 아니면 0을 출력합니다.
3. **64비트 바이너리 해시**가 생성됩니다 (8행 × 8비교).
4. **해시 비교**: 두 해시 간의 **해밍 거리** (다른 비트 수)가 지각적 차이를 측정합니다.

**임계값 해석:**

| 임계값 | 의미 |
|--------|------|
| 0 | 정확한 지각적 일치 (동일한 이미지) |
| 1~5 | 매우 유사 (미세한 압축 아티팩트, 약간의 크롭) |
| 6~10 | 유사 (작은 편집, 색상 조정) — **기본값: 10** |
| 11~20 | 어느 정도 유사 (눈에 띄는 차이) |
| 21~64 | 점점 더 다름 |

**픽셀 비교 대신 dHash를 사용하는 이유:**
- 리사이즈, 미세한 크롭, 압축 아티팩트에 강건함
- 매우 빠름: 해싱 후 비교당 O(1)
- 서로 다른 이미지 형식(JPEG vs PNG)에서도 작동

**ssook에서 사용하는 방법:**
1. **Data** 탭 → **Near-Duplicate Detector**로 이동
2. 이미지 디렉토리 설정
3. 임계값 설정 (기본값: 10)
4. **Run** 클릭 — 중복 쌍이 해밍 거리와 함께 나열됩니다
