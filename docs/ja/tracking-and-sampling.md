## Object Tracking（オブジェクトトラッキング）

オブジェクトトラッキングは、動画フレーム間で検出されたオブジェクトに一貫したIDを割り当てます。ssookは、任意の検出モデルの上で動作する2つのトラッキングアルゴリズムを提供します。

### SORT

**Simple Online and Realtime Tracking**

SORTはシンプルなIoUベースのトラッカーです：

1. **検出**：検出モデルが現在のフレームのバウンディングボックスを生成。
2. **IoUマッチング**：既存の各トラックの最後の既知位置と各新規検出間のIoU（重なり）を計算。M×Nのコスト行列が生成されます。
3. **貪欲割り当て**：最も高いIoUのペアから順にトラックと検出をマッチング。マッチにはIoU ≥ `iou_threshold`（デフォルト：0.3）が必要。
4. **マッチしたトラックの更新**：トラックの位置を更新、ヒットカウントを増加、未更新時間をリセット。
5. **新規トラックの作成**：マッチしなかった検出は新しいIDで新規トラックになります。
6. **マッチしなかったトラックのエイジング**：このフレームでマッチしなかったトラックは未更新時間が増加。`max_age` フレームを超えてマッチしないトラックは削除されます。
7. **出力**：合計 `min_hits` 回以上マッチしたトラックのみが報告されます（単一フレーム検出によるちらつきを防止）。

各トラックは**軌跡** — 直近60個の中心点 — を保持し、可視化に使用されます。

### ByteTrack

ByteTrackは、**2段階アソシエーション**を使用して低信頼度の検出を回復することでSORTを改善します：

**ステージ1 — 高信頼度マッチング：**
1. 検出を**高信頼度**（スコア ≥ `high_thresh`、デフォルト：0.5）と**低信頼度**（スコア ≥ `low_thresh`、デフォルト：0.1）に分割。
2. SORTと同様に、すべての既存トラックを高信頼度検出とIoUでマッチング。
3. マッチしたトラックを更新、マッチしなかったトラックを収集。

**ステージ2 — 低信頼度回復：**
4. **残りのマッチしなかったトラック**を低信頼度検出とIoUでマッチング。
5. これにより、一時的にオクルージョンされたり検出信頼度が低かったトラックが回復されます。
6. 両ステージ後もマッチしなかったトラックは未更新時間が増加。

**新規トラック作成：**
7. マッチしなかった**高信頼度**検出のみが新規トラックを作成。低信頼度検出だけでは新規トラックを開始しません（ノイズ防止）。

**ByteTrackが優れている理由：**
- 標準的なトラッカーは低信頼度検出を完全に破棄するため、オブジェクトが部分的にオクルージョンされるとトラックが断片化します。
- ByteTrackは低信頼度検出を使用して既存トラックを維持しますが、新規トラックの作成には使用しません（False Positiveを回避）。

### パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|---------|-------------|
| `max_age` | 30 | マッチなしでトラックを維持するフレーム数 |
| `min_hits` | 3 | トラックが報告されるまでの最小マッチ数 |
| `iou_threshold` | 0.3 | 有効なマッチの最小IoU |
| `high_thresh` | 0.5 | ByteTrack：第1段階マッチングの最小信頼度 |
| `low_thresh` | 0.1 | ByteTrack：第2段階マッチングの最小信頼度 |

**ssookでの使い方：**
1. **Viewer** タブに移動
2. 検出モデルを読み込み、動画を開く
3. トラッキングを有効にし、アルゴリズム（ByteTrackまたはSORT）を選択
4. オブジェクトに永続的なIDと軌跡トレイルが割り当てられます

## Smart Sampler（スマートサンプラー）

大規模データセットを扱う際、代表的なサブセットが必要になることがよくあります。Smart Samplerは3つの戦略を提供します。

### Balanced Sampling（バランスサンプリング）

**目標**：すべてのクラスの均等な表現と、各クラス内の空間的多様性。

**仕組み：**
1. すべてのラベルファイルを解析し、クラス → 画像のマップを構築。
2. ターゲット数をクラス間で均等に分割：`per_class = target / num_classes`。
3. 各クラスについて、プールが `per_class` より大きい場合、**最遠点サンプリング**を使用：
   - 各画像の特徴量を計算：平均バウンディングボックス中心 `[cx, cy]`（0〜1に正規化）。
   - ランダムなシード画像から開始。
   - 既に選択されたすべての画像から**最も遠い**画像を反復的に選択（ユークリッド距離の二乗を使用）。
   - これにより空間的多様性が確保されます — 選択された画像は一箇所に集中するのではなく、画像空間の異なる領域をカバーします。
4. プールが `per_class` より小さい場合、そのクラスのすべての画像を使用。

**使用場面**：クラス不均衡があり均等な表現が必要な場合、または多様な空間カバレッジが必要な場合。

### Stratified Sampling（層化サンプリング）

**目標**：元のクラス分布をより小さなスケールで維持。

**仕組み：**
1. 各クラスの比率を計算：`class_count / total_associations`。
2. 各クラスから `proportion × target` 枚の画像をランダムにサンプリング。
3. 合計がターゲットに満たない場合、未選択プールからランダムに画像を追加して残りを埋める。

**使用場面**：元の分布を統計的に代表する小さなデータセットが必要な場合。

### Random Sampling（ランダムサンプリング）

**目標**：シンプルで偏りのないランダムサブセット。

**仕組み**：再現性のために固定シードで `random.sample(all_images, target)` を実行。

**使用場面**：クラスバランスが重要でない場合、またはベースラインとして。

**ssookでの使い方：**
1. **Data** タブ → **Smart Sampler** に移動
2. 画像ディレクトリ、ラベルディレクトリ、出力ディレクトリを設定
3. 戦略（Balanced / Stratified / Random）を選択
4. ターゲット数とランダムシードを設定
5. **Run** をクリック — サンプリング前後のクラス分布テーブルが表示されます

## Near-Duplicate Detection（類似重複検出）

類似重複検出は、**dHash**（差分ハッシュ）を使用して、知覚的に類似した画像（バイト単位で同一とは限らない）を見つけます。

**dHashの仕組み：**

1. 画像を9×8ピクセル（グレースケール）に**リサイズ**。高周波の詳細を除去し、サイズを正規化します。
2. **水平勾配を計算**：各行で隣接ピクセルを比較。左のピクセルが右より明るければ1、そうでなければ0を出力。
3. **64ビットのバイナリハッシュ**（8行 × 8比較）が生成されます。
4. **ハッシュの比較**：2つのハッシュ間の**ハミング距離**（異なるビット数）が知覚的な差異を測定します。

**閾値の解釈：**

| 閾値 | 意味 |
|-----------|---------|
| 0 | 完全な知覚的一致（同一画像） |
| 1〜5 | 非常に類似（軽微な圧縮アーティファクト、わずかなクロップ） |
| 6〜10 | 類似（小さな編集、色調整）— **デフォルト：10** |
| 11〜20 | やや類似（目立つ違い） |
| 21〜64 | 徐々に異なる |

**ピクセル比較ではなくdHashを使う理由：**
- リサイズ、軽微なクロップ、圧縮アーティファクトに対してロバスト
- 非常に高速：ハッシュ後は比較あたりO(1)
- 異なる画像フォーマット（JPEG vs PNG）間で動作

**ssookでの使い方：**
1. **Data** タブ → **Near-Duplicate Detector** に移動
2. 画像ディレクトリを設定
3. 閾値を設定（デフォルト：10）
4. **Run** をクリック — 重複ペアがハミング距離とともにリストされます
