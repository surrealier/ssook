## Object Tracking（目标跟踪）

目标跟踪为视频帧中检测到的目标分配一致的 ID。ssook 提供两种跟踪算法，可与任何检测模型配合使用。

### SORT

**Simple Online and Realtime Tracking（简单在线实时跟踪）**

SORT 是一种基于 IoU 的简洁跟踪器：

1. **检测**：检测模型为当前帧生成边界框。
2. **IoU 匹配**：计算每个现有轨迹的最后已知位置与每个新检测之间的 IoU（重叠度），生成 M×N 代价矩阵。
3. **贪心分配**：优先选择 IoU 最高的配对进行匹配。匹配要求 IoU ≥ `iou_threshold`（默认：0.3）。
4. **更新已匹配轨迹**：更新轨迹位置，增加命中计数，重置未更新时间。
5. **创建新轨迹**：未匹配的检测成为新轨迹，分配新 ID。
6. **老化未匹配轨迹**：本帧未匹配的轨迹，其未更新时间递增。超过 `max_age` 帧未匹配的轨迹将被移除。
7. **输出**：只有总匹配次数 ≥ `min_hits` 的轨迹才会被报告（防止单帧检测导致的闪烁）。

每条轨迹维护一个**轨迹线**——最近 60 个中心点——用于可视化。

### ByteTrack

ByteTrack 通过**两阶段关联**改进了 SORT，能够恢复低置信度检测：

**第一阶段——高置信度匹配：**
1. 将检测分为**高置信度**（分数 ≥ `high_thresh`，默认：0.5）和**低置信度**（分数 ≥ `low_thresh`，默认：0.1）。
2. 使用 IoU 将所有现有轨迹与高置信度检测进行匹配，方式与 SORT 相同。
3. 更新已匹配轨迹；收集未匹配轨迹。

**第二阶段——低置信度恢复：**
4. 使用 IoU 将**剩余未匹配轨迹**与低置信度检测进行匹配。
5. 这可以恢复暂时被遮挡或检测置信度较低的轨迹。
6. 两个阶段后仍未匹配的轨迹，其未更新时间递增。

**新轨迹创建：**
7. 只有未匹配的**高置信度**检测才会创建新轨迹。低置信度检测不会单独启动新轨迹（防止噪声）。

**为什么 ByteTrack 更好：**
- 标准跟踪器完全丢弃低置信度检测，导致目标被部分遮挡时轨迹断裂。
- ByteTrack 利用低置信度检测维持现有轨迹，但不让它们创建新轨迹（避免误检）。

### 跟踪参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_age` | 30 | 无匹配时保持轨迹存活的帧数 |
| `min_hits` | 3 | 轨迹被报告前所需的最少匹配次数 |
| `iou_threshold` | 0.3 | 有效匹配的最小 IoU |
| `high_thresh` | 0.5 | ByteTrack：第一阶段匹配的最低置信度 |
| `low_thresh` | 0.1 | ByteTrack：第二阶段匹配的最低置信度 |

**在 ssook 中使用：**
1. 进入 **Viewer** 标签页
2. 加载检测模型并打开视频
3. 启用跟踪并选择算法（ByteTrack 或 SORT）
4. 目标将被分配持久 ID 并显示轨迹线

## Smart Sampler（智能采样器）

处理大型数据集时，通常需要一个具有代表性的子集。智能采样器提供三种策略。

### Balanced Sampling（均衡采样）

**目标**：所有类别均等表示，且每个类别内具有空间多样性。

**工作原理：**
1. 解析所有标签文件，构建类别 → 图像的映射。
2. 将目标数量均分到各类别：`per_class = target / num_classes`。
3. 对每个类别，如果候选池大于 `per_class`，使用**最远点采样**：
   - 为每张图像计算特征：平均边界框中心 `[cx, cy]`（归一化到 0–1）。
   - 从随机种子图像开始。
   - 迭代选择与所有已选图像**距离最远**的图像（使用欧氏距离平方）。
   - 这确保了空间多样性——选中的图像覆盖图像空间的不同区域，而非聚集在一处。
4. 如果候选池小于 `per_class`，则取该类别的所有图像。

**适用场景**：类别不平衡且需要均等表示时，或需要多样化空间覆盖时。

### Stratified Sampling（分层采样）

**目标**：以较小规模保持原始类别分布。

**工作原理：**
1. 对每个类别，计算其比例：`class_count / total_associations`。
2. 从每个类别中随机采样 `proportion × target` 张图像。
3. 如果总数少于目标数，从未选中的候选池中随机补充。

**适用场景**：需要一个在统计上代表原始分布的较小数据集时。

### Random Sampling（随机采样）

**目标**：简单的无偏随机子集。

**工作原理**：使用固定种子的 `random.sample(all_images, target)` 以确保可复现性。

**适用场景**：类别平衡不重要时，或作为基线。

**在 ssook 中使用：**
1. 进入 **Data** 标签页 → **Smart Sampler**
2. 设置图像目录、标签目录和输出目录
3. 选择策略（Balanced / Stratified / Random）
4. 设置目标数量和随机种子
5. 点击 **Run**——显示采样前后的类别分布对比表

## Near-Duplicate Detection（近似重复检测）

近似重复检测使用 **dHash**（差异哈希）查找感知上相似（但不一定字节相同）的图像。

**dHash 工作原理：**

1. 将图像**缩放**到 9×8 像素（灰度）。这去除了高频细节并归一化了尺寸。
2. **计算水平梯度**：对每行，比较相邻像素。如果左侧像素比右侧亮，输出 1；否则输出 0。
3. 生成一个 **64 位二进制哈希**（8 行 × 8 次比较）。
4. **比较哈希**：两个哈希之间的 **Hamming 距离**（不同位的数量）衡量感知差异。

**阈值解读：**

| 阈值 | 含义 |
|------|------|
| 0 | 完全感知匹配（相同图像） |
| 1–5 | 非常相似（轻微压缩伪影、微小裁剪） |
| 6–10 | 相似（小幅编辑、颜色调整）——**默认值：10** |
| 11–20 | 有一定相似性（明显差异） |
| 21–64 | 差异逐渐增大 |

**为什么选择 dHash 而非像素比较：**
- 对缩放、轻微裁剪和压缩伪影具有鲁棒性
- 极快：哈希后每次比较 O(1)
- 跨不同图像格式有效（JPEG vs PNG）

**在 ssook 中使用：**
1. 进入 **Data** 标签页 → **Near-Duplicate Detector**
2. 设置图像目录
3. 设置阈值（默认：10）
4. 点击 **Run**——列出重复对及其 Hamming 距离
